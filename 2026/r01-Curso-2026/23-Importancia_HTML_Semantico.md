# Importancia del HTML Semántico

## ¿Por qué deberías preocuparte por HTML semántico?

El significado semántico de un elemento se refiere a qué información especial transmite ese elemento.

La mayoría de los elementos tienen significado semántico. El elemento div es uno de los pocos que no lo tiene.


## ¿Por qué es importante tener una buena jerarquía estructural?

Puede ser tentador usar un elemento de encabezado específico debido a su estilo, como `h1` para texto grande.
En su lugar, deberías elegir el elemento apropiado para la estructura de tu documento y usar CSS para lograr el estilo que te gustaría.

Usar la jerarquía correcta es importante para la accesibilidad.

Usar un elemento `h3` después de un `h1` puede causar que un usuario de lector de pantalla crea que accidentalmente se ha saltado contenido importante, debido a la falta de un elemento `h2`.


## ¿Cuál es la diferencia entre HTML presentacional y HTML semántico?

El tamaño y color de la fuente siempre deben establecerse en CSS, no en HTML.

Los elementos de HTML presentacional ya están obsoletos y ya no son recomendados. El HTML semántico es ahora la práctica recomendada. Describe el contenido de los elementos, por lo que es mucho más fácil de leer, entender y mantener.

Ahora sabes la diferencia entre HTML presentacional y HTML semántico: HTML semántico describe el contenido, mientras que HTML presentacional se enfoca en la apariencia.
